한국어 정책

Flat Earth Disc 사이트가 현재 어떤 정보를 어떤 목적으로 처리하는지 요약합니다.

1. 사이트가 직접 수집하는 정보

이 사이트는 회원가입, 로그인, 댓글, 주문, 결제, 직접 입력 폼을 제공하지 않습니다. 따라서 이름, 이메일, 전화번호 같은 직접 식별 정보를 이 페이지에서 별도로 입력받아 저장하지 않습니다.

다만 웹사이트에 접속하면 일반적인 웹 전달 과정에서 호스팅 또는 CDN 제공자가 IP 주소, 브라우저 정보, 요청 시간, 리퍼러 같은 접속 로그를 일시적으로 처리할 수 있습니다. 이 정보는 서비스 제공, 보안, 장애 대응 목적에 사용될 수 있습니다.

2. 브라우저 로컬 저장소

이 사이트는 사용자 환경을 유지하기 위해 브라우저의 localStorage를 사용합니다.

flat-earth-language 언어 설정 유지
flat-earth-layout-mode HUD / Classic 레이아웃 선택 유지
flat-earth-onboarding-v1 온보딩 안내를 다시 표시하지 않기 위한 상태 저장
flat-earth-detail-collapsed 상세 패널 접힘 상태 유지

이 값들은 기본적으로 사용자의 브라우저 안에만 저장되며, 사용자가 사이트 데이터를 지우기 전까지 남아 있을 수 있습니다.

3. 업로드한 지도 파일 처리

메인 뷰어는 사용자가 업로드한 SVG 지도 파일을 브라우저 안에서 즉시 미리보기용으로 처리합니다. 현재 코드 기준으로 이 업로드는 URL.createObjectURL() 방식으로 로컬에서 처리되며, 이 페이지 자체가 해당 파일을 별도 서버로 업로드하도록 구성되어 있지 않습니다.

4. 광고 및 동의 관리

메인 뷰어는 Google AdSense 광고 태그를 로드합니다. 사용자의 지역, 브라우저 설정, 동의 상태, Google의 광고 운영 방식에 따라 Google 및 관련 파트너가 쿠키, 로컬 저장소, IP 주소, 기기 또는 브라우저 식별자, 대략적 위치 정보, 광고 측정 또는 사기 방지 관련 정보를 처리할 수 있습니다.

또한 지원 지역에서는 Google Privacy & Messaging / CMP 메시지가 표시될 수 있으며, 메인 뷰어의 설정 패널에서 광고 동의 선택을 다시 열 수 있습니다.

5. 외부 서비스 및 외부 요청

  • 메인 뷰어는 사이트 자체 정적 파일과 JSON 데이터를 불러옵니다.
  • 메인 뷰어는 Google Fonts를 사용하므로 글꼴 로딩 과정에서 Google 서버와 연결될 수 있습니다.
  • 광고 또는 동의 인터페이스가 로드될 때 Google 관련 도메인과 네트워크 통신이 발생할 수 있습니다.

이러한 요청 과정에서 외부 서비스 제공자는 IP 주소, 브라우저 헤더, 요청 URL, 리퍼러 등의 기술 정보를 처리할 수 있습니다.

6. 보관 기간

  • 브라우저 로컬 저장소 값은 사용자가 직접 삭제하거나 사이트가 값을 덮어쓸 때까지 유지될 수 있습니다.
  • 업로드한 SVG 미리보기는 현재 브라우저 세션과 탭 컨텍스트에서만 사용됩니다.
  • 광고 또는 동의 관련 데이터의 보관 기간은 Google 및 관련 파트너 정책에 따를 수 있습니다.
  • 접속 로그 보관 기간은 실제 호스팅/인프라 제공자의 운영 정책에 따라 달라질 수 있습니다.

7. 이용자의 선택권

  • 브라우저 설정에서 쿠키와 사이트 데이터를 차단하거나 삭제할 수 있습니다.
  • 브라우저 개발자 도구 또는 사이트 데이터 삭제 기능으로 localStorage 값을 제거할 수 있습니다.
  • 메인 뷰어 설정 패널에서 Google 광고 동의 선택창을 다시 열 수 있습니다.
  • 시크릿 모드 또는 별도 프로필을 사용하면 로컬 저장소 분리를 도울 수 있습니다.

English Policy

This section mirrors the same technical behavior summary in English.

1. Information collected directly by this site

This site does not currently provide account registration, login, comments, checkout, or direct contact forms. As a result, it does not intentionally ask visitors to submit direct identifiers such as name, email address, or phone number through this page.

During ordinary website delivery, the hosting platform or CDN may temporarily process technical access data such as IP address, browser metadata, request time, and referrer for service delivery, security, and troubleshooting.

2. Browser local storage

The main viewer uses localStorage to preserve interface preferences.

flat-earth-language Stores the selected language.
flat-earth-layout-mode Stores the selected HUD or Classic layout.
flat-earth-onboarding-v1 Stores whether the onboarding callout has already been dismissed.
flat-earth-detail-collapsed Stores whether the detail panel is collapsed.

These values are stored in the visitor's browser and may remain until the visitor clears site data or the site overwrites them.

3. Uploaded map file handling

The main viewer lets visitors preview an uploaded SVG map locally in the browser. Based on the current code, that preview uses URL.createObjectURL() and is not intentionally configured to upload the selected file to a separate server endpoint.

4. Advertising and consent

The main viewer loads a Google AdSense tag. Depending on region, browser settings, consent choice, and Google's ad-serving flow, Google and related partners may process cookies, local storage values, IP address, browser or device identifiers, approximate location, referrer data, and ad measurement or fraud-prevention data.

In supported regions, Google Privacy & Messaging / CMP prompts may also be shown. Visitors can revisit those ad privacy choices from the viewer settings panel.

5. External services and requests

  • The main viewer loads same-origin static assets and JSON data.
  • The main viewer uses Google Fonts, which may require a network request to Google servers.
  • Google ad or consent interfaces may trigger network requests to Google domains.

Those requests may expose technical data such as IP address, browser headers, requested URL, and referrer to the relevant service provider.

6. Retention

  • Browser storage persists until the visitor clears it or the site replaces it.
  • Uploaded SVG previews are used in the local browser context for the current session/tab.
  • Advertising and consent data retention may be governed by Google and partner policies.
  • Server-side log retention depends on the actual hosting or infrastructure provider in use.

7. Choices available to visitors

  • Visitors can block or delete cookies and site data through browser settings.
  • Visitors can remove localStorage values by clearing site data.
  • Visitors can reopen Google's ad consent choices from the main viewer settings panel.
  • Private browsing or a separate browser profile can help isolate local browser storage.
